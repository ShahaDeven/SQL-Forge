# SQLForge SFT Dataset Card

Synthetic supervised-fine-tuning data for Text-to-SQL over the modified TPC-H (SF=0.1)
DuckDB schema. Built to teach the house-style conventions the base Qwen2.5-Coder-7B
missed in Phase 0 (canonical customer→region joins, `total_value*(1-promo_reduction)`
revenue, real categorical literals, column-format simulations).

## Summary

| | |
|---|---|
| **Total validated pairs** | **2,436** |
| Train / Val | 2,314 / 122 (95/5, stratified by tier) |
| Format | chat JSONL (`system` = schema + house rules [+ sim contract], `user` = question, `assistant` = SQL) |
| Held-out test | the 55-query suite in `eval/benchmark.json` — **never** in training |

Weighted toward the tiers where the base 7B was weakest in Phase 0 (aggregation,
multi-hop, window, simulation).

## Provenance

- **Questions:** generated by `claude-haiku-4-5` with a revenue-framing guard and
  per-tier simulation contract.
- **Gold SQL (labels):** `claude-haiku-4-5` with the house-style system prompt +
  keyword few-shot + a self-healing retry loop (feeds DB errors back). Sonnet 5 was
  validated as a labeler but Haiku was chosen for budget (~10× cheaper) — quality is
  enforced by the gate, not raw model size.
- **Split by source:** 1,373 generated + 1,063 recovered. The recovered pairs had been
  dropped by an over-strict Haiku self-consistency check (which rejected ~38% of *valid*
  pairs); they were rescued via `recover_consistency.py` after re-passing the gate, at
  zero additional API cost. The consistency check is now off by default.

## Quality gate (every pair passes both)

1. **Execution** — runs read-only on TPC-H DuckDB within a 5s watchdog timeout; rejects
   syntax/exec errors, empty results, single-null cells, and >50k-row (cartesian) results.
2. **House-style** — rejects superseded revenue columns (`l_extendedprice`/`l_discount`)
   and non-canonical categorical literals.

## Contamination

`contamination.py` checks question 4-gram and normalized-SQL 5-gram + AST overlap of
every training pair against the eval suite. See `contamination_report.json`.

- **True contamination (question + SQL duplicate of an eval query): 0**
- Max question 4-gram Jaccard: 0.667 (shared common words, no duplicate questions)
- 57 benign SQL-only overlaps, all on trivial tiers (simple/join/aggregation) where a
  query like `SELECT COUNT(*) FROM customer` has a single canonical form — not leakage.

## Reproduce

```bash
python sqlforge/data/generate_sft.py --total 2000   # generate (Haiku, gate on)
python sqlforge/data/recover_consistency.py          # rescue over-rejected pairs
python sqlforge/data/split.py                         # 95/5 stratified split
python sqlforge/data/contamination.py                # evidence report
```
